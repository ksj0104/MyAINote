# NULLSEC OS — 데스크톱 셸 재설계 (Design Spec)

날짜: 2026-06-27
대상 프로젝트: TERMINAL//BREACH (정적 브라우저 해킹 교육 게임)
상태: 승인됨 (브레인스토밍 완료) → 구현 계획 단계로 진행

## 1. 개요 / 목표

페이지에 접속하면 **가상 컴퓨터의 바탕화면(데스크톱 OS)** 이 나타나고, 플레이어는
바탕화면 아이콘을 클릭해 **앱(메신저·터미널·브라우저)** 을 창으로 띄워 미션을 푼다.
"실제 컴퓨터를 쓰는 것처럼" 느껴지는 몰입형 셸로 기존의 패널/툴독/메뉴 화면을 대체한다.

핵심 원칙:
- **게임 엔진은 그대로 재사용한다.** 미션 판정·명령 인터프리터·대화 엔진을 건드리지 않는다.
- **셸(표현/조작 레이어)만 교체한다.** 창 시스템 + 3개 앱 + 데스크톱.
- 모든 앱 동작은 결국 기존 `game.exec(...)` 경로로 수렴한다 → **playtest 105개 그대로 통과**.

비목표(Out of scope):
- 모바일 최적화(데스크톱 우선; 좁은 화면은 최소 동작만).
- 미션 내용/정답/스토리 변경(이번 작업은 셸 교체에 한정).
- 멀티플레이·서버·계정(정적 사이트 유지, 저장은 localStorage).

## 2. 재사용 vs 교체 (경계)

**그대로 재사용 (수정 최소):**
- `js/commands.js` — 명령 인터프리터(99+ 명령)
- `js/levels.js` — 34미션(interlude/brief/objective/goal/hints/setup/check/success/stance)
- `js/comms.js` — 0xMOTHER/WRAITH 대화 엔진(chat/channel/contacts)
- `js/filesystem.js` — 가상 파일시스템
- `js/game.js` — 핵심 로직: `exec`, `loadLevel`, `markCleared`, 판정, STANCE, 저장/불러오기

**교체/신규 (셸):**
- `index.html`, `css/style.css` — 데스크톱 셸로 재작성
- `js/terminal.js` 의 **렌더링/메뉴/패널/툴독 부분 제거** → 데스크톱·앱 모듈로 이전
- 기존 메뉴 화면(타이틀 히어로·menu-cards), 좌우 패널(fstree/targets/story-panel), 하단 command-bar,
  방금 추가한 툴독(`#tool-dock`) → **전부 제거**, 데스크톱 메타포로 대체

**UI 브리지:** `game.js` 가 호출하던 UI 메서드(예: `renderStory`, `appendStory`, `setMission`,
`println`, `clear`, `setMode`, `showMenu`, `showEnding`, `showMissionSelect` 등)는 **같은 이름의
표면(인터페이스)을 유지**하되, 내부에서 데스크톱 앱으로 라우팅하도록 재구현한다.
(`window.term` 표면을 유지하거나 `js/os/ui-bridge.js`로 분리. game.js 변경은 호출부 연결만 최소.)

## 3. 부팅 & 세션 런처

1. 접속 → 짧은 **부팅 시퀀스**(NULLSEC OS 로고/부팅 로그, 1~2초, 클릭/키로 스킵 가능).
2. **바탕화면** 표시(월페이퍼 + 앱 아이콘 + 작업표시줄). 아직 플레이 불가 상태.
3. 화면 중앙에 **세션 런처 모달**(프로그램 창 형태):
   - `▶ 새 게임` — 진행도 초기화 후 L01부터.
   - `⏵ 이어하기` — 저장 있을 때만 활성, 저장 지점부터.
   - 닫기 전까지 바탕화면 조작은 흐릿하게 비활성(모달 포커스).
4. 선택 시 모달 닫힘 → 데스크톱에서 플레이 시작(첫 브리핑 메신저 도착).
5. 시작버튼(작업표시줄)에서 언제든 런처/설정 재호출 가능.

## 4. 창 시스템 (윈도우 매니저) — `js/os/window-manager.js`

- **창 모델:** `{ id, appKey, title, icon, el(content), x,y,w,h, z, minimized, maximized }`
- **조작:**
  - 타이틀바 드래그로 이동(pointer 이벤트, 화면 경계 클램프)
  - 우하단 핸들로 리사이즈(최소 크기 제한)
  - 타이틀바 버튼 `─ □ ✕` = 최소화 / 최대화(워크스페이스 채움 토글) / 닫기
  - 창 클릭 시 포커스(z-index 최상단), 포커스 창만 강조 테두리
- **단일 인스턴스:** 앱당 창 1개(이미 열려 있으면 포커스/복원).
- **작업표시줄 연동:** 열림/최소화 창을 버튼으로 표시, 클릭 토글.
- **API(예):** `wm.open(appKey)`, `wm.focus(id)`, `wm.close(id)`, `wm.toggleMin(id)`, `wm.isOpen(appKey)`.

## 5. 데스크톱 — `js/os/desktop.js`

- **월페이퍼:** 다크 네이비/네온 시안, 은은한 그리드/스캔라인.
- **앱 아이콘(3종):** 📨 메신저 · 💻 터미널 · 🌐 브라우저 (클릭=실행/포커스). 키보드 접근 가능.
- **세션 런처 모달**(§3) 호출/관리.
- **미션 위젯(상시 표시, §6):** 바탕화면 우상단 고정.
- **작업표시줄(하단):** 시작버튼(NULLSEC 로고 → 런처·설정·도움말 팝업) + 열린 창 버튼 + 시계.
- **알림:** 새 메신저 메시지 도착 시 메신저 아이콘/작업표시줄 버튼 깜빡임(뱃지).

## 6. 미션 위젯 (데스크톱 상시)

- 표시: **현재 미션 ID·제목 / 한 줄 목표(objective) / 진행도 n/34 / STANCE 게이지(신념↔효율)**.
- "지금 할 일"은 항상 위젯으로 확인(브리핑 본문/대사는 메신저). 길 잃음 방지.
- 미션 전환 시 갱신. 클릭하면 메신저의 해당 브리핑으로 점프(선택적).

## 7. 앱 사양

### 7.1 메신저 — `js/apps/messenger.js`
- 좌: **연락처 목록**(0xMOTHER 기본, ACT II/L14부터 WRAITH 추가 — `game.knownContacts` 기반).
- 우: **대화 스레드**(채널별), 말풍선 + **타이핑 효과**(기존 메신저 큐 로직 이전).
- **미션 브리핑/디브리핑이 여기로 도착**(0xMOTHER/WRAITH 대사). 기존 `renderStory`/`appendStory`
  내용이 메신저 스레드로 렌더된다.
- 하단 입력창으로 `chat` 가능(엔진 `comms` 재사용), `channel`/`contacts` 동작.
- 새 메시지 알림(아이콘 깜빡임). 브리핑 도착 시 메신저 자동 포커스(또는 알림만 — 설정 가능).

### 7.2 터미널 — `js/apps/terminal-app.js`
- 명령 콘솔(주력). 입력 → `game.exec(raw)` → 출력 로그(타자 효과 유지), 히스토리(↑↓), Tab 자동완성.
- 프롬프트는 현재 user@host:cwd 반영. 셸 미션 전부 여기서 해결.
- 사운드/블립 효과 유지(설정에서 토글).

### 7.3 브라우저 — `js/apps/browser.js`
- **주소창 + 페이지 뷰포트 + 새로고침/뒤로**. 수준: *주소창 뷰어 + 간단 폼*.
- URL 이동 = 내부적으로 `curl <url>` 실행 → 결과를 **웹페이지처럼 렌더**.
- **페이지 타입별 렌더:**
  - 텍스트/robots.txt 등: 모노스페이스 텍스트로 표시(`game.web` 내용).
  - **로그인 페이지**: 아이디/비번 폼 + 로그인 버튼. 비번칸에 SQLi 페이로드 입력 → `login admin "<payload>"` 실행(S2.02/2.03). 폼 아래 "쿼리 미리보기"로 학습 보조(선택).
  - **다운로드/LFI**: 주소창 `?file=...` 파라미터 이동 = `curl ...?file=...`(S2.04). 결과 텍스트 표시.
  - **업로드 페이지**: 파일 선택(준비된 shell.php 등) + 업로드 버튼 → `curl -X POST -F ...`(S2.06).
- 페이지 정의는 미션 상태(`game.web`, `game.uploadEndpoint`, 로그인 엔드포인트 등)에서 파생.
  알 수 없는 URL은 "연결할 수 없음" 페이지. **모든 동작은 기존 엔진 명령으로 수렴**(검증 로직 불변).

> 주력은 터미널이며, 웹 미션은 브라우저로도/터미널 `curl`로도 풀 수 있다(둘 다 같은 엔진 경로).

## 8. 미션 흐름

1. `game.loadLevel(i)` → 브리핑(0xMOTHER 대사)이 **메신저 스레드**로 도착(타이핑) + **위젯 목표 갱신**
   + 새 메시지 알림. (기존 잠금/게이팅 개념은 "브리핑 도착 알림"으로 대체.)
2. 플레이어가 터미널/브라우저로 작업 → `exec` → `check` 통과 시 클리어.
3. 0xMOTHER **디브리핑**(메신저) → `advance()` → 다음 브리핑 도착 → 위젯 갱신.
4. 분기 미션(S3.02/S3.08)·엔딩 동일. **엔딩**은 전체화면 테이크오버(또는 전용 창)로 STANCE 3종 분기 연출.

## 9. 비주얼 — NULLSEC OS (다크 사이버 해커 OS)

- 팔레트: 다크 네이비 + 네온 시안(+ 그린/앰버 테마는 설정에서 전환, 기존 토큰 일부 계승).
- 폰트: 본문/콘솔 `VT323`(모노), UI/타이틀 `Chakra Petch`.
- 창: 글로우 타이틀바, 코너 브래킷/네온 보더, 은은한 CRT/스캔라인.
- 일관성: 모든 창·위젯·작업표시줄이 동일 디자인 언어.

## 10. 설정 / 저장

- 설정 접근: 작업표시줄 시작버튼 팝업(또는 메신저/터미널 명령). 항목: 테마, 사운드,
  저장(`save`)/불러오기(`load`)/슬롯, 처음부터(`restart`), 런처로.
- 저장은 기존 localStorage 스키마 유지(이어하기 = 기존 save 사용).

## 11. 파일 구조 (예정)

```
index.html              데스크톱 셸 마크업(부팅·바탕화면·작업표시줄·런처 컨테이너)
css/style.css           NULLSEC OS 스타일(창/아이콘/위젯/작업표시줄/앱)
js/filesystem.js        (유지)
js/commands.js          (유지)
js/levels.js            (유지)
js/comms.js             (유지)
js/game.js              (유지 — UI 호출부만 브리지로 연결)
js/os/window-manager.js (신규) 창 생성·드래그·리사이즈·포커스·작업표시줄
js/os/desktop.js        (신규) 바탕화면·아이콘·런처·미션위젯·알림
js/os/ui-bridge.js      (신규) game.js가 기대하는 UI 표면 → 앱 라우팅
js/apps/messenger.js    (신규) 교신/브리핑 앱
js/apps/terminal-app.js (신규) 명령 콘솔 앱
js/apps/browser.js      (신규) 웹 미션 브라우저 앱
playtest.js             (유지)
```

(기존 `js/terminal.js`는 부팅/사운드/메신저 큐/엔진 호출 같은 재사용 로직만 추출해
위 모듈들로 이전하고, 패널·메뉴·툴독 코드는 폐기한다.)

## 12. 테스트 / 검증

- **playtest.js (105 검사) 그대로 통과** — `game.exec`/`check` 무변경이 전제. 회귀 가드.
- 브라우저 스모크(Playwright): 부팅→런처(새 게임)→데스크톱, 창 열기/드래그/포커스/최소화/닫기,
  메신저 브리핑 도착, 터미널로 미션 1개 클리어, 브라우저 로그인 폼으로 S2.02 클리어, 위젯 갱신.
- 콘솔 에러 0 확인.

## 13. 마이그레이션 메모

- 점진 전환이 어려우므로 셸 교체는 하나의 작업 흐름으로 진행하되, 각 단계에서 playtest를 통과시켜
  엔진 무결성을 보장한다.
