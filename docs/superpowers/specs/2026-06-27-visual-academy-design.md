# 시각 학습 모드 (Visual Academy) — Design Spec

날짜: 2026-06-27
대상 프로젝트: TERMINAL//BREACH · NULLSEC OS (정적 브라우저 해킹 교육 게임)
상태: 승인됨 (브레인스토밍 완료) → 구현 계획 단계로 진행
선행 문서: [NULLSEC OS 데스크톱 셸 재설계](2026-06-27-nullsec-os-desktop-design.md)

## 1. 개요 / 목표

기존 텍스트 전용 학습 모드(`js/modes.js`의 Academy — `lessons`/`learn <명령>` 텍스트 카드)를
**3분할 전체화면 시각 코스**로 교체한다. `그 외 모드 → 학습(ACADEMY)` 진입 시 전체화면 학습
화면이 뜨고, 69개 명령을 **카테고리 순서대로 한 개씩** 배운다.

각 명령마다 **① 설명 → ② 애니메이션 시각화 → ③ 직접 실습**을 거치며,
**실습을 통과해야 다음 명령이 잠금 해제**된다. 명령이 "어떻게 동작했는지"를 무대에서
시각적으로(파일트리·주소 스캔·침투 토폴로지·바이트 변환 등) 보여주는 것이 핵심.

핵심 원칙:
- **게임 엔진은 그대로 재사용한다.** 명령 인터프리터·판정을 건드리지 않는다.
  실습 콘솔 입력은 전부 기존 `game.exec(...)` 경로로 수렴 → playtest 회귀 영향 없음.
- **학습 모드(Academy)만 교체한다.** 본편 스토리 미션 화면·로직은 손대지 않는다.
- 기존 학습 텍스트 자료(`DETAILS`/`EXAMPLES`/`COMMAND_OPTS`)를 **레슨 본문으로 재사용**한다.

## 2. 비목표 (Out of scope / YAGNI)

- 본편 스토리/미션 화면·판정 변경 (학습 모드 한정).
- 명령 인터프리터(`js/commands.js`)·정답 판정 로직 변경.
- 모바일 최적화(데스크톱 우선). 신규 사운드 제작(기존 블립만 재사용).
- 69개 외 신규 명령 추가.

## 3. 진입 / 통합 방식

- 진입점: 기존 메뉴 **`그 외 모드 → 1. 학습(ACADEMY)`** 를 유지하되, 진입 시
  텍스트 출력 대신 **전체화면 학습 화면(takeover overlay)** 을 띄운다.
- 전체화면 오버레이는 데스크톱/창 위에 덮이며, 상단바의 `✕` 또는 `menu` 입력으로 닫고
  타이틀 메뉴로 복귀한다.
- 학습 모드의 샌드박스 파일시스템/유저(`student@academy`) 초기화는 기존 `Academy.enter`
  로직을 계승한다(자유 실습용 안전 FS).

## 4. 화면 레이아웃 (3분할 — 브레인스토밍 확정안 B)

```
┌─────────────────────────────────────────────────┐
│ ◈ ACADEMY               진도 12/69   [✕ 닫기/menu] │  상단바
├──────────┬──────────────────────────────────────┤
│ 커리큘럼  │  [설명 스트립]  nmap — 포트·서비스 스캔  │
│ 사이드바  ├──────────────────────────────────────┤
│ ▸네트워크 │                                       │
│  ✔ ping  │        시각화 무대 (애니메이션)         │
│  ▶ nmap  │     (파일트리/스캔/토폴로지/XOR…)       │
│  🔒 ssh  │                                       │
│ ▸암호    ├──────────────────────────────────────┤
│  🔒 …    │ student@academy:~$ nmap 10.0.0.0/24 ▍ │  실습 콘솔
└──────────┴──────────────────────────────────────┘
```

- **커리큘럼 사이드바(좌):** 9개 카테고리 + 명령별 상태 아이콘
  (`✔` 완료 / `▶` 현재 / `🔒` 잠김). 상단/하단에 진도율(n/69). 완료·현재 명령은 클릭으로 점프 가능.
- **설명 스트립(우상단):** 현재 명령 이름 + 한 줄 설명(분류·사용법·옵션은 펼치기).
- **시각화 무대(우중앙):** 명령 설명 시 **자동 데모 재생**, 실습 시 **친구가 입력한 명령에
  반응하는 실시간 애니메이션**. 패밀리별 렌더러가 그린다(§5).
- **실습 콘솔(우하단):** 프롬프트 + 입력. 실제 `game.exec()` 실행, 히스토리(↑↓)·Tab 자동완성·
  타자 효과 유지. 통과 판정도 여기서.

## 5. 시각화 패밀리 (5 정교 + 1 라이트)

각 명령은 정확히 하나의 패밀리에 속한다. 패밀리 렌더러는 `(명령, 인자, 샌드박스 상태)`를 받아
애니메이션을 재생한다. 스타일: **픽셀/CRT 네온**(게임 본편과 동일 언어, 다크 네이비 + 네온 시안,
기존 3색 테마 토큰 계승). 구현은 주로 CSS keyframe + DOM, 토폴로지/연결선은 SVG.

| # | 패밀리 | 명령 | 애니메이션 내용 |
|---|--------|------|----------------|
| ① | 파일시스템 | `ls cd cat find grep mkdir touch cp mv rm echo pwd` | 디렉토리 트리 렌더 — 커서가 트리를 훑고(강조 스윕), `find`/`grep` 일치 노드가 시안색으로 켜짐, `cd` 현재 위치 이동 강조, `mkdir`/`touch` 새 노드 생성, `rm` 노드 소멸, `cat` 파일 내용 펼침 |
| ② | 네트워크 | `nmap ping ssh scp netstat ifconfig arp route wget curl exit disconnect` | 호스트 토폴로지 — `nmap` 스캔 라인이 훑으며 살아있는 호스트가 켜지고 열린 포트 표시, `ssh` 침투 시 그 노드까지 HOP 연결선이 굵어지고 컨텍스트가 원격으로 이동, `ping` 왕복 핑 |
| ③ | 암호·인코딩 | `base64 rot13 caesar xor vigenere md5sum strings xxd` | 입력 바이트/글자가 키와 결합(시프트·XOR·치환)되어 평문으로 변하는 변환 흐름. `strings`/`xxd`는 바이너리에서 가독 문자열/헥스 추출 |
| ④ | 공격·크래킹 | `hydra john hashcat dump` | 워드리스트가 빠르게 흐르다 일치 비번에 걸려 멈춤(하이라이트), 해시→평문 매칭, `dump`는 DB에서 자격증명 추출 흐름 |
| ⑤ | 무선 | `airmon-ng airodump-ng aireplay-ng aircrack-ng` | 모니터 모드 진입 → AP↔클라이언트 토폴로지 → deauth 패킷 발사 → 핸드셰이크 캡처 → WPA 키 크랙 진행바 |
| ⑥ | 상태(라이트) | `whoami id chmod su sudo passwd ps top kill uname env export df free uptime date history clear man help` + `chat reply say channel contacts` + `submit save load saves` | 가벼운 상태 시각화: 권한 비트 `rwx` 토글, 유저 배지(guest→root) 변화, 프로세스/시스템 리스트 패널, 또는 단순 정보 카드 |

> 모든 69개 명령은 **최소한 ⑥ 라이트 시각 반응**을 갖는다. ①~⑤ 패밀리는 전용 정교 애니메이션.
> 명령→패밀리 매핑은 신규 데이터 테이블(`VIZ_FAMILY`)로 관리.

## 6. 진행 / 잠금 (실습 게이팅)

1. 현재 명령의 설명 표시 + 무대에서 **데모 자동재생**(대표 인자 사용).
2. 친구가 콘솔에 **그 명령을 직접 입력** → `game.exec` 실행 → 무대가 실시간 반응 + 콘솔 출력.
3. 해당 명령을 **올바른 형태로 1회 실행**하면 `✔완료` 처리, 다음 명령 잠금 해제 +
   "다음 ▶"으로 진행(자동 이동 또는 버튼).
   - "올바른 형태" 판정: 입력 토큰의 첫 명령어가 현재 학습 대상과 일치하고, 명령이
     에러 없이(또는 의미 있는 출력으로) 실행되었는지로 가볍게 판정. 완벽한 결과를 요구하지 않음.
4. 완료 상태는 localStorage에 저장(기존 `codelabSolved` 패턴 재사용 — `academyDone: Set<command>`).
   이미 끝낸 명령은 사이드바에서 자유롭게 다시 점프해 복습 가능.
5. 막힐 때: 콘솔에서 `man <명령>`·힌트("이렇게 쳐보세요" 예시)·`learn <명령>` 항상 사용 가능.
6. 자유 실습: 현재 명령 외 다른 명령도 콘솔에서 자유롭게 실행 가능(샌드박스). 잠금은
   "다음 레슨 진입"에만 적용되고 실습 자체를 막지 않는다.

## 7. 구현 구조 (파일)

```
js/apps/academy.js     (신규) 학습 코스 컨트롤러 — 진도/잠금/레슨 라우팅/콘솔 입력 처리
js/os/academy-viz.js   (신규) 5+1 패밀리 렌더러 — (명령,인자,상태)→애니메이션 재생
js/modes.js            (수정) Academy.enter 를 전체화면 학습 화면 띄우기로 교체.
                              기존 DETAILS/EXAMPLES/CATEGORIES/COMMAND_OPTS 재사용.
                              신규 VIZ_FAMILY(명령→패밀리) 매핑 추가.
index.html             (수정) 학습 오버레이 컨테이너 마크업 + 신규 스크립트 로드
css/style.css          (수정) 학습 전용 클래스(.acad-*, 패밀리 애니메이션 keyframe). 3색 테마 토큰 계승.
js/game.js             (영향 없음) exec/판정 경로 그대로. 실습 콘솔이 game.exec 호출.
playtest.js            (유지) 기존 105 검사 회귀 가드. 필요 시 학습 진입 스모크 1건 추가.
```

## 8. 데이터 모델

- `VIZ_FAMILY`: `{ [command]: 'fs'|'net'|'crypto'|'crack'|'wifi'|'light' }` — 신규.
- 레슨 본문: 기존 `DETAILS[cmd]`(desc/ex/tip/see) + `EXAMPLES[cmd]` + `COMMAND_OPTS[cmd]` 재사용.
- 데모 인자: 패밀리 렌더러가 명령별 대표 인자를 내장(예: `nmap 10.0.0.0/24`, `find / -name "*.key"`).
- 진도 저장: `game.academyDone`(Set, localStorage 직렬화). 기존 save 스키마에 키 1개 추가.

## 9. 테스트 / 검증

- **playtest.js (기존 105 검사) 그대로 통과** — `game.exec`/`check` 무변경이 전제. 회귀 가드.
- 학습 진입 스모크: `그 외 모드 → 학습` 진입 시 오버레이가 뜨고 첫 명령(ls 등) 레슨이
  렌더되는지, 콘솔에서 해당 명령 실행 시 완료 처리·잠금 해제되는지 1건 검사 추가.
- 브라우저 스모크(Playwright): 학습 오버레이 열기 → 사이드바 표시 → 명령 입력 →
  무대 애니메이션 DOM 갱신 → 다음 잠금 해제. 콘솔 에러 0 확인.

## 10. 시각 스타일 (확정)

- 픽셀/CRT 네온 — 다크 네이비(`#070b16`/`#0a0e1a`) + 네온 시안(`#16e0c8`/`#5fd7ff`),
  매치/하이라이트는 시안 글로우, 경고/포트는 앰버(`#ffd166`). 폰트 `VT323`.
- 기존 3색 테마(amber/green/cyan) 토큰을 계승해 `theme` 전환과 일관 동작.
- 애니메이션: 스캔 스윕·노드 팝·바이트 변환·진행바 등 CSS keyframe 중심, 토폴로지 연결선은 SVG.
